# 🔴 Phase 1: Fix Commission System (Confirmed Bugs + New Foundation)

## ⚠️ ONE BUSINESS DECISION NEEDED BEFORE STARTING

Currently the code charges platform commission on the FULL order total (food + delivery).
The recommended fix below charges platform commission ONLY on the delivery company's
delivery revenue (delivery_fee + restaurant_delivery_subsidy), NOT on food price.

**This matches how `global_commission_delivery` is named** (a "delivery" commission, 
parallel to `global_commission_ride`, `global_commission_shipment`) and is the standard 
model used by real-world delivery aggregators (they don't take a cut of food price, 
only of the logistics fee).

**If your actual business model is "platform takes % of total order value including food"**,
tell the developer to keep using `params.total` instead of `deliveryRevenueBase` in Step 2 below.
Everything else in this prompt works identically either way.

---

## CONFIRMED BUGS (verified by reading actual code)

1. `createOrderFromCart` in `restaurantApi.ts` (line ~258) uses **hardcoded `commissionRate = 12`** 
   — completely ignores admin's `accounting_settings.global_commission_delivery` and any 
   `partner_commission_settings` override.
2. The restaurant is **never recorded as a financial party** — `financial_transactions` only 
   has one row per order, attributed entirely to `delivery_company_id`. Restaurant's earnings 
   are invisible to the platform-level ledger.
3. `restaurants.commission_rate` (set in `DeliveryRestaurants.tsx` dashboard) is **saved but 
   never read in any calculation** anywhere in the codebase.
4. `DeliveryFinance.tsx` restaurant settlement calculation (`totalRevenue = totalFoodRevenue - totalSubsidy`) 
   **does not subtract `commission_rate`** — and the Supabase query for restaurants on line ~95 
   doesn't even select that column.
5. `getPartnerCommissionSettings` / `upsertPartnerCommission` functions exist in `accountingApi.ts` 
   but **no admin UI page calls them** — there is no way to actually set a partner override today.
6. No function exists to create `partner_invoices` rows — invoices can only be viewed/updated, 
   never generated.

## STEP 2 — Fix `createOrderFromCart` to split revenue 3 ways

File: `artifacts/wasal/src/lib/restaurantApi.ts`

Replace the entire commission/financial_transactions block (currently around line 258-289):

```typescript
const insertData: any = {
  // ... unchanged, keep exactly as is ...
};
if (params.notes) insertData.notes = params.notes;
if (params.applied_offer_id) insertData.applied_offer_id = params.applied_offer_id;

const { data, error } = await supabase.from("delivery_orders").insert(insertData).select().single();
if (error) throw error;

// ──────────────────────────────────────────────────────────────
// Financial split — THREE parties: platform / delivery company / restaurant
// ──────────────────────────────────────────────────────────────
try {
  const { calculateCommission } = await import("./accountingApi");

  // 1. Delivery company's gross delivery revenue
  //    = what customer paid for delivery + what restaurant owes for subsidized free delivery
  const subsidy = params.restaurant_delivery_subsidy ?? 0;
  const deliveryRevenueBase = params.delivery_fee + subsidy;

  // 2. Platform commission comes out of the delivery company's delivery revenue
  const { commission: platformCommission, earning: companyDeliveryEarning } =
    await calculateCommission(deliveryRevenueBase, "delivery", params.delivery_company_id);

  // 3. Restaurant's net food revenue
  //    = food subtotal - delivery company's cut (commission_rate) - subsidy owed for free delivery
  const { data: restRow } = await supabase
    .from("restaurants")
    .select("commission_rate, name_ar")
    .eq("id", params.restaurant_id)
    .maybeSingle();
  const restCommissionRate = Number((restRow as any)?.commission_rate || 0);
  const restCommissionCut = Math.floor(params.subtotal * restCommissionRate / 100);
  const restaurantNetEarning = params.subtotal - restCommissionCut - subsidy;

  // Fetch delivery company name for denormalized partner_name
  const { data: companyRow } = await supabase
    .from("delivery_companies")
    .select("company_name")
    .eq("user_id", params.delivery_company_id)
    .maybeSingle();

  // Row 1 — Delivery company settlement (platform's commission ledger)
  await supabase.from("financial_transactions").insert({
    reference_id: data.id,
    order_id: data.id,
    transaction_type: "delivery_order",
    partner_type: "delivery_company",
    customer_id: params.customer_id,
    partner_id: params.delivery_company_id,
    partner_name: (companyRow as any)?.company_name ?? null,
    amount: deliveryRevenueBase,
    platform_commission: platformCommission,
    partner_earning: companyDeliveryEarning,
    payment_method: params.payment_method,
    payment_status: "pending",
    notes: `عمولة توصيل — طلب ${data.id}`,
  });

  // Row 2 — Restaurant settlement (no platform commission on this leg by default,
  // since the platform's cut already came from the delivery company's leg above)
  await supabase.from("financial_transactions").insert({
    reference_id: data.id,
    order_id: data.id,
    transaction_type: "restaurant_order",
    partner_type: "restaurant",
    customer_id: params.customer_id,
    partner_id: params.restaurant_id,
    partner_name: (restRow as any)?.name_ar ?? null,
    amount: params.subtotal,
    platform_commission: 0,
    partner_earning: restaurantNetEarning,
    payment_method: params.payment_method,
    payment_status: "pending",
    notes: `صافي إيرادات الطعام — طلب ${data.id} (عمولة شركة التوصيل: ${restCommissionCut} ر.ي)`,
  });
} catch (finErr) {
  // Never block order creation if financial tracking fails — log only
  console.error("Financial transaction split failed:", finErr);
}

// Clear cart
await clearCart(params.customer_id, params.restaurant_id);
return data;
```

}

const AdminCommissionOverrides = () => {
  const [partners, setPartners] = useState<PartnerRow[]>([]);
  const [loading, setLoading] = useState(true);
  const [editing, setEditing] = useState<Record<string, { type: string; value: number; on: boolean }>>({});
  const [saving, setSaving] = useState<string | null>(null);

  useEffect(() => {
    const load = async () => {
      // Adjust table/column names below to match your actual delivery_companies / suppliers schema
      const { data: companies } = await supabase.from("delivery_companies").select("user_id, company_name");
      const { data: suppliers } = await supabase.from("suppliers").select("user_id, company_name").then(r => r).catch(() => ({ data: [] as any[] }));

      const withOverrides = await Promise.all(

                      className="w-24"
                    />
                  </>
                )}

                <Button size="sm" onClick={() => handleSave(p.id)} disabled={saving === p.id}>
                  {saving === p.id ? "..." : "حفظ"}
                </Button>
              </CardContent>
            </Card>

        })}

## STEP 5 — Build invoice generation function

Add to `artifacts/wasal/src/lib/accountingApi.ts`:

```typescript
export const createPartnerInvoice = async (
  partnerId: string,
  partnerType: string,
  periodStart: string,
  periodEnd: string
) => {
  const { data: txns, error } = await supabase
    .from("financial_transactions")
    .select("*")
    .eq("partner_id", partnerId)
    .gte("created_at", periodStart)
    .lte("created_at", periodEnd)
    .is("paid_at", null);

  if (error) throw error;
  if (!txns || txns.length === 0) return null;

  const totalAmount = txns.reduce((s, t) => s + Number(t.amount), 0);
  const totalCommission = txns.reduce((s, t) => s + Number(t.platform_commission), 0);
  const netAmount = txns.reduce((s, t) => s + Number(t.partner_earning), 0);

  const invoiceNumber = `INV-${partnerId.slice(0, 6).toUpperCase()}-${periodStart.slice(0, 7)}`;

  return supabase.from("partner_invoices").insert({
    partner_id: partnerId,
    invoice_number: invoiceNumber,
    period_start: periodStart,
    period_end: periodEnd,
    period_type: "monthly",
    total_amount: totalAmount,
    total_commission: totalCommission,
    net_amount: netAmount,
    total_transactions: txns.length,
    due_date: new Date(new Date(periodEnd).getTime() + 7 * 24 * 60 * 60 * 1000).toISOString(),
    status: "pending",
  }).select().single();
};

export const generateMonthlyInvoicesForAllPartners = async () => {
  const now = new Date();
  const periodStart = new Date(now.getFullYear(), now.getMonth() - 1, 1).toISOString();
  const periodEnd = new Date(now.getFullYear(), now.getMonth(), 0).toISOString();

  const { data: partnerIds } = await supabase
    .from("financial_transactions")
    .select("partner_id, partner_type")
    .gte("created_at", periodStart)
    .lte("created_at", periodEnd)
    .is("paid_at", null);

  const uniquePartners = [...new Map((partnerIds || []).map(p => [p.partner_id, p])).values()];

  const results = [];
  for (const p of uniquePartners) {
    const result = await createPartnerInvoice(p.partner_id, (p as any).partner_type, periodStart, periodEnd);
    if (result) results.push(result);
  }
  return results;
};
```

## STEP 6 — Schedule monthly invoice generation (Vercel Cron)

Create `api/invoices/generate-monthly.ts`:

```typescript
import type { VercelRequest, VercelResponse } from '@vercel/node';
import { createClient } from '@supabase/supabase-js';

export default async function handler(req: VercelRequest, res: VercelResponse) {
  // Protect with a secret so only Vercel Cron can call this
  const authHeader = req.headers.authorization;
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return res.status(401).json({ error: 'Unauthorized' });
  }

  const supabaseUrl = process.env.SUPABASE_URL ?? '';
  const serviceKey = process.env.SUPABASE_SERVICE_ROLE_KEY ?? '';
  const supabase = createClient(supabaseUrl, serviceKey);

  const now = new Date();
  const periodStart = new Date(now.getFullYear(), now.getMonth() - 1, 1).toISOString();
  const periodEnd = new Date(now.getFullYear(), now.getMonth(), 0).toISOString();

  const { data: txns } = await supabase
    .from('financial_transactions')
    .select('partner_id, partner_type, amount, platform_commission, partner_earning')
    .gte('created_at', periodStart)
    .lte('created_at', periodEnd)
    .is('paid_at', null);

  const grouped: Record<string, { amount: number; commission: number; earning: number; count: number; type: string }> = {};
  for (const t of (txns || [])) {
    if (!grouped[t.partner_id]) grouped[t.partner_id] = { amount: 0, commission: 0, earning: 0, count: 0, type: t.partner_type };
    grouped[t.partner_id].amount += Number(t.amount);
    grouped[t.partner_id].commission += Number(t.platform_commission);
    grouped[t.partner_id].earning += Number(t.partner_earning);
    grouped[t.partner_id].count += 1;
  }

  const created = [];
  for (const [partnerId, g] of Object.entries(grouped)) {
    const invoiceNumber = `INV-${partnerId.slice(0, 6).toUpperCase()}-${periodStart.slice(0, 7)}`;
    const { data } = await supabase.from('partner_invoices').insert({
      partner_id: partnerId,
      invoice_number: invoiceNumber,
      period_start: periodStart,
      period_end: periodEnd,
      period_type: 'monthly',
      total_amount: g.amount,
      total_commission: g.commission,
      net_amount: g.earning,
      total_transactions: g.count,
      due_date: new Date(now.getTime() + 7 * 24 * 60 * 60 * 1000).toISOString(),
      status: 'pending',
    }).select().single();
    if (data) created.push(data);
  }

  return res.status(200).json({ generated: created.length, invoices: created });
}
```

Add to `vercel.json` (root):
```json

    { "path": "/api/invoices/generate-monthly", "schedule": "0 3 1 * *" }

```
