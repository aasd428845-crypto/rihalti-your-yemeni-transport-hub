1.1. جعل زر "موافقة" يعمل بشكل صحيح

في ملف src/pages/delivery/FinancePage.tsx (أو ما يعادلها)، تأكد من أن دالة handleApprove تقوم بالآتي:

```tsx
const handleApprove = async (paymentTxId: string) => {
  try {
    // 1. جلب بيانات المعاملة من payment_transactions مع تفاصيل العميل
    const { data: paymentTx, error: fetchError } = await supabase
      .from('payment_transactions')
      .select(`
        *,
        customer:user_id ( full_name, phone )
      `)
      .eq('id', paymentTxId)
      .single();
    if (fetchError) throw fetchError;

    // 2. تحديث حالة payment_transactions إلى 'verified'
    await supabase
      .from('payment_transactions')
      .update({ status: 'verified', verified_by: user.id, verified_at: new Date() })
      .eq('id', paymentTxId);

    // 3. تحديث financial_transactions المرتبطة إلى 'paid'
    await supabase
      .from('financial_transactions')
      .update({ payment_status: 'paid' })
      .eq('payment_transaction_id', paymentTxId);

    // 4. إرسال إشعار للعميل
    await supabase.functions.invoke('send-push-notification', {
      body: {
        userId: paymentTx.user_id,
        title: 'تم تأكيد الدفع',
        body: `تم قبول دفعتك بقيمة ${paymentTx.amount} ريال. شكراً لثقتك.`,
      },
    });

    // 5. تحديث الواجهة (إعادة جلب البيانات)
    toast.success('تمت الموافقة على المعاملة');
    fetchTransactions(); // دالة تعيد جلب البيانات
  } catch (error) {
    toast.error('حدث خطأ في الموافقة');
  }
};
```

رابعاً: إصلاح نظام روابط الدعوة للشركاء والمندوبين

4.1. التحقق من جدول invitation_tokens

تأكد من وجود الجدول بالهيكل التالي:

```sql
CREATE TABLE IF NOT EXISTS invitation_tokens (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  email TEXT NOT NULL,
  role TEXT NOT NULL CHECK (role IN ('supplier', 'delivery', 'driver', 'delivery_driver')),
  token TEXT UNIQUE NOT NULL,
  expires_at TIMESTAMPTZ NOT NULL,
  used_at TIMESTAMPTZ,
  created_by UUID REFERENCES auth.users(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

4.2. تحديث واجهة توليد الرابط في لوحة التحكم

في لوحة تحكم المشرف، يجب أن تكون صفحة "الدعوات" تحتوي على:

· اختيار الدور (صاحب مكتب، شركة توصيل، سائق أجرة، مندوب توصيل).
· إدخال البريد الإلكتروني.
· زر "توليد رابط".
· بعد التوليد، يظهر الرابط الكامل (مثل https://wasl.app/invite/{token}) مع زر نسخ.

كود مثال للنموذج:

```tsx
const generateInvite = async (email, role) => {
  const token = crypto.randomUUID();
  const expiresAt = new Date();
  expiresAt.setDate(expiresAt.getDate() + 7); // صلاحية 7 أيام

  const { error } = await supabase
    .from('invitation_tokens')
    .insert({
      email,
      role,
      token,
      expires_at: expiresAt.toISOString(),
      created_by: user.id,
    });
  if (error) throw error;

  const inviteLink = `${window.location.origin}/invite/${token}`;
  setGeneratedLink(inviteLink);
};
```

4.3. التأكد من أن صفحة InvitePage تعمل لجميع الأدوار

افتح ملف src/pages/InvitePage.tsx (الموجود مسبقاً) وتأكد من أنه:

· يتحقق من صحة التوكن.
· بناءً على الدور (من التوكن)، يعرض نموذج إنشاء الحساب المناسب:
  · لصاحب مكتب (supplier): حقول إضافية (اسم المكتب، شعار، حساب بنكي، مناطق عمل).
  · لشركة توصيل (delivery): حقول إضافية (اسم الشركة، شعار، مناطق تغطية).
  · لسائق أجرة (driver): حقول متقدمة (صورة بطاقة، سيلفي، رخصة، معلومات مركبة).
  · لمندوب توصيل (delivery_driver): حقول (الاسم، الهاتف، نوع المركبة، رقم اللوحة، رقم الهوية، صورة البطاقة، رخصة).
· بعد التقديم، ينشئ المستخدم في auth.users مع الدور المناسب ويربطه بالشريك (إذا كان مندوباً، يربطه بشركة التوصيل التي أنشأت الرابط).

4.4. ربط مندوب التوصيل بشركة التوصيل

عند إنشاء حساب مندوب توصيل، يجب إضافة delivery_company_id إلى جدول delivery_drivers (أو إلى profiles كحقل إضافي). هذا الربط هو ما يسمح لشركة التوصيل برؤية مندوبيها وإسناد الطلبات إليهم.

5.2. إسناد طلب لمندوب (بضغطة زر)

عند ظهور طلب جديد (من جدول delivery_orders)، يجب أن يكون هناك زر "تعيين مندوب" يفتح قائمة بالمندوبين المتاحين (مع حالتهم وتقييمهم). بعد اختيار مندوب، يتم تحديث delivery_orders بالحقول:

```tsx
await supabase
  .from('delivery_orders')
  .update({ driver_id: selectedDriverId, status: 'assigned', assigned_at: new Date() })
```

بعد الإسناد، يتم إرسال إشعار فوري للمندوب (باستخدام OneSignal).

5.3. تحديث حالة الطلب من قبل المندوب

في تطبيق المندوب، يجب أن يكون هناك أزرار لتغيير الحالة (استلمت، في الطريق، تم التسليم). كل تغيير يحدث في قاعدة البيانات ويرسل إشعاراً للشركة والعميل.

5.4. عرض الطلبات في لوحة شركة التوصيل

صفحة "الطلبات" يجب أن تعرض جميع الطلبات مع حالتها والمندوب المسند إليها (إن وجد). مع إمكانية إعادة الإسناد إذا لم يقبل المندوب.

سادساً: اختبار شامل للتأكد من عمل كل شيء

بعد تنفيذ جميع التعديلات، يجب اختبار السيناريوهات التالية:

1. سير عمل الدفع:
   · عميل يدفع عبر تحويل بنكي → تظهر المعاملة في لوحة شركة التوصيل مع صورة الحوالة.
   · مشرف/شركة تضغط "موافقة" → تتغير الحالة ويصل إشعار للعميل.
2. سير عمل الدعوات:
   · مشرف ينشئ رابط دعوة لمندوب توصيل → يرسل الرابط للمندوب.
   · مندوب يفتح الرابط ويملأ النموذج → ينشئ حسابه ويرتبط تلقائياً بالشركة.
3. سير عمل الطلبات:
   · عميل يطلب توصيلاً من مطعم → يظهر الطلب في لوحة الشركة.
   · الشركة تسند الطلب لمندوب → يصل إشعار للمندوب.
   · المندوب ينفذ الطلب ويغير الحالة → تنعكس الحالة في لوحة الشركة وعند العميل.
4. الأداء والتجاوب:
   · التبديل بين الصفحات سريع (لا يزيد عن ثانيتين).
   · الصفحات المالية تظهر بشكل جيد على الجوال.
