الأمر الشامل لتطوير نظام المطاعم والمنيو في منصة "وصل" – إضافة الصور وتحسين الواجهة

تعليمات هامة للذكاء الاصطناعي:
هذا الأمر يهدف إلى إكمال وتحسين نظام المطاعم والمنيو بناءً على الملاحظات الواردة في الصورة. يجب تنفيذه بعد إكمال الأوامر السابقة (نظام مناطق التغطية وطلبات التسعيرة). ركز على إضافة الصور وتحسين واجهة عرض المطاعم والمنيو لتكون أكثر جاذبية واحترافية، مع إعطاء شركة التوصيل القدرة على إدارة كل هذه التفاصيل من لوحة التحكم.

---

الجزء الأول: تحديث قاعدة البيانات – إضافة حقول الصور والتفاصيل

1.1. إضافة حقول جديدة إلى جدول restaurants

تأكد من وجود الحقول التالية (أضف المفقود):

```sql
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS cover_image_url TEXT; -- صورة الغلاف (الخلفية الكبيرة للمطعم)
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS logo_url TEXT; -- شعار المطعم (الصورة المصغرة)
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS description TEXT; -- وصف المطعم
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS cuisine_type TEXT[]; -- أنواع المطبخ (يمني، مشروبات، ...)
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS rating DECIMAL(3,2) DEFAULT 0;
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS total_ratings INTEGER DEFAULT 0;
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS min_order_amount DECIMAL(10,2) DEFAULT 0;
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS delivery_fee DECIMAL(10,2) DEFAULT 0;
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS estimated_delivery_time INTEGER; -- بالدقائق
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS is_active BOOLEAN DEFAULT true;
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS is_featured BOOLEAN DEFAULT false; -- للمطاعم المميزة
ALTER TABLE restaurants ADD COLUMN IF NOT EXISTS opening_hours JSONB; -- ساعات العمل (مثلاً {"saturday": "9am-11pm"})
```

1.2. تحديث جدول menu_items – إضافة تفاصيل أكثر للصنف

أضف الحقول التالية إلى جدول menu_items (إذا لم تكن موجودة):

```sql
ALTER TABLE menu_items ADD COLUMN IF NOT EXISTS image_url TEXT; -- صورة الطعام
ALTER TABLE menu_items ADD COLUMN IF NOT EXISTS description TEXT; -- وصف تفصيلي
ALTER TABLE menu_items ADD COLUMN IF NOT EXISTS calories INTEGER; -- السعرات الحرارية
ALTER TABLE menu_items ADD COLUMN IF NOT EXISTS ingredients TEXT[]; -- المكونات (مصفوفة نصوص)
ALTER TABLE menu_items ADD COLUMN IF NOT EXISTS is_available BOOLEAN DEFAULT true; -- هل الصنف متاح حالياً؟
ALTER TABLE menu_items ADD COLUMN IF NOT EXISTS is_featured BOOLEAN DEFAULT false; -- صنف مميز
ALTER TABLE menu_items ADD COLUMN IF NOT EXISTS sort_order INTEGER DEFAULT 0; -- ترتيب العرض
ALTER TABLE menu_items ADD COLUMN IF NOT EXISTS preparation_time INTEGER; -- وقت التحضير بالدقائق
```

1.3. إنشاء جدول الخيارات الإضافية للأصناف (اختياري) – إذا لم يكن موجوداً

```sql
CREATE TABLE IF NOT EXISTS menu_item_options (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  menu_item_id UUID REFERENCES menu_items(id) ON DELETE CASCADE,
  name_ar TEXT NOT NULL, -- اسم الخيار (مثلاً "الحجم")
  option_type TEXT CHECK (option_type IN ('size', 'extra', 'remove')), -- حجم، إضافة، استبعاد
  choices JSONB, -- مثال: [{"name_ar": "كبير", "price": 5}, {"name_ar": "وسط", "price": 0}]
  is_required BOOLEAN DEFAULT false,
  max_selections INTEGER DEFAULT 1,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

الجزء الثاني: تطوير لوحة تحكم شركة التوصيل – إدارة المطاعم والمنيو المتقدمة

2.1. تحديث صفحة إضافة/تعديل المطعم

· في ملف src/pages/delivery/restaurants/AddRestaurant.tsx (أو EditRestaurant.tsx)، أضف حقول رفع الصور:
  · صورة الغلاف (cover image) – حقل رفع ملف، يُرفع إلى Supabase Storage في مجلد restaurant-covers.
  · شعار المطعم (logo) – يُرفع إلى restaurant-logos.
  · جميع الحقول الأخرى المذكورة (الوصف، أنواع المطبخ، ساعات العمل، الحد الأدنى للطلب، رسوم التوصيل، وقت التوصيل التقديري).
· استخدم نموذج إدخال متعدد الخطوات أو صفحة واحدة مع معاينة للصورة.

2.2. تطوير صفحة إدارة المنيو (Menu Management)

· في صفحة تفاصيل المطعم، أضف زر "إدارة المنيو" ينتقل إلى صفحة MenuPage تعرض:
  · فئات المنيو (كما هي موجودة حالياً) مع إمكانية إضافة فئة جديدة (اسم، صورة فئة).
  · عند اختيار فئة، تظهر الأصناف التابعة لها.
· لكل صنف، اعرض في بطاقة:
  · صورة الطعام (إذا وجدت) مع إمكانية تغييرها.
  · اسم الصنف، السعر، حالة التوفر.
  · أزرار تعديل/حذف.
· أضف زر "إضافة صنف جديد" يفتح نموذجاً يحتوي على:
  · اسم الصنف (عربي وإنجليزي اختياري).
  · وصف الصنف (textarea).
  · السعر (رقم إلزامي).
  · صورة الطعام (رفع ملف).
  · السعرات الحرارية (رقم اختياري).
  · المكونات (حقل نصي يمكن إدخال عدة مكونات مفصولة بفواصل).
  · وقت التحضير (بالدقائق، اختياري).
  · حالة التوفر (checkbox).
  · خيار "صنف مميز" (يظهر في أعلى القائمة).
  · الخيارات الإضافية: زر "إضافة خيار" يسمح بإنشاء خيارات مثل الأحجام أو الإضافات (يتم تخزينها في menu_item_options). يجب أن يتضمن كل خيار:
    · اسم الخيار (مثل "الحجم").
    · نوع الخيار (حجم، إضافة، استبعاد).
    · قائمة الخيارات المتاحة (مع أسعارها).
    · هل هو إجباري؟
    · الحد الأقصى للاختيار (مثلاً 1 للحجم).
· بعد الحفظ، يتم تحديث menu_items و menu_item_options.

2.3. إضافة معاينة حية للمنيو

· في صفحة إدارة المنيو، أضف زر "معاينة المطعم" يفتح صفحة المطعم كما سيراها العميل (في نافذة جديدة أو نفس الصفحة) ليرى كيف ستبدو الصور والتفاصيل.

الجزء الثالث: تحسين واجهة العميل – عرض المطاعم والمنيو

3.1. تحديث صفحة قائمة المطاعم (/restaurants أو صفحة التوصيل الرئيسية)

· اعرض المطاعم في بطاقات أنيقة تحتوي على:
  · صورة الغلاف (cover_image_url) كخلفية للبطاقة مع تأثير ضبابي في الأعلى.
  · شعار المطعم (logo_url) بشكل دائري.
  · اسم المطعم، أنواع المطبخ، التقييم (نجوم).
  · الحد الأدنى للطلب، رسوم التوصيل، وقت التوصيل.
  · زر "عرض المنيو".
· اجعل المطاعم المميزة (is_featured = true) تظهر في قسم منفصل "الأكثر طلباً" مع تنسيق مميز.

3.2. إنشاء صفحة تفاصيل المطعم (/restaurant/:id)

· في أعلى الصفحة، اعرض:
  · صورة الغلاف كخلفية للهيدر.
  · شعار المطعم أسفل الغلاف مع اسم المطعم والتقييم.
  · معلومات سريعة: الحد الأدنى، رسوم التوصيل، وقت التوصيل، ساعات العمل.
· عرض فئات المنيو: يمكن عرضها كأزرار (أو قائمة جانبية) للتنقل السريع.
· عرض الأصناف:
  · قسّم الأصناف حسب الفئة.
  · لكل صنف، بطاقة تحتوي على:
    · صورة الطعام (إذا وجدت) – على اليمين أو اليسار حسب التصميم.
    · اسم الصنف، وصف مختصر، السعر.
    · إذا كان للصنف خيارات (أحجام، إضافات)، أضف زر "تخصيص" أو اعرضها في نافذة منبثقة.
    · زر "إضافة إلى السلة".
· عند النقر على صنف، افتح نافذة منبثقة (Modal) تحتوي على:
  · صورة أكبر للطعام.
  · اسم الصنف ووصف كامل.
  · الخيارات المتاحة (أحجام، إضافات) مع إمكانية الاختيار وتحديث السعر.
  · حقل الكمية.
  · زر "إضافة إلى السلة".

3.3. إضافة سلة الطلبات (Cart)

· يمكن عرض السلة في صفحة منفصلة أو في شريط جانبي.
· تعرض الأصناف المضافة مع الخيارات والكمية والسعر الإجمالي.
· إمكانية تعديل الكمية أو حذف الصنف.
· عرض إجمالي السلة ورسوم التوصيل.
· زر "متابعة الطلب" ينتقل إلى صفحة إتمام الطلب.

3.4. صفحة إتمام الطلب (/checkout)

· تعرض ملخص السلة.
· اختيار عنوان التوصيل (من العناوين المحفوظة أو إضافة جديد).
· اختيار وقت التوصيل (فوري أو تحديد وقت).
· طريقة الدفع (نقدي، تحويل بنكي).
· زر "تأكيد الطلب" ينشئ الطلب في delivery_orders ويخزن payment_transactions حسب طريقة الدفع.

الجزء الرابع: تحسين الأداء والتجاوب

· استخدم React.lazy و Suspense لتحميل صفحات المطاعم والمنيو بشكل غير متزامن.
· أضف loading="lazy" للصور.
· استخدم useMemo و useCallback لتجنب إعادة التصيير غير الضرورية.
· تأكد من أن التصميم متجاوب مع الجوال (استخدم Tailwind classes مثل grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3).

---

الجزء الخامس: اختبار شامل

بعد تنفيذ جميع الخطوات، اختبر السيناريوهات التالية:

1. إضافة مطعم جديد من لوحة التحكم مع رفع صورة غلاف وشعار، وإدخال جميع التفاصيل.
2. إضافة فئات وأصناف مع صور وخيارات إضافية.
3. عرض المطعم في واجهة العميل – تأكد من ظهور الصور بشكل صحيح والتفاصيل.
4. إضافة صنف إلى السلة مع اختيار خيارات إضافية، وتأكيد السعر الصحيح.
5. إتمام الطلب واختيار طريقة الدفع.
6. ظهور الطلب في لوحة تحكم شركة التوصيل مع إمكانية قبوله وتعيين مندوب.

   - استخدم Lazy Loading للصور.
